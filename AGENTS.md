## Quick Style Reference

**Voice:**
- Conversational expertise (colleague-to-colleague, not condescending)
- Show your thinking process, including discoveries and mistakes
- Humble confidence: deep knowledge + acknowledgment of limitations
- Ground advice in specific personal/professional experiences

**Post Length:**
- Target: 5-7 minute read (~1,000-1,400 words)
- For longer topics: create a series instead of one long post

**Common Structures:**
- Framework/Concept posts: Problem → Personal context → Framework → Application → Conclusion
- Technical How-To posts: Context → Why-it-matters → Prerequisites → Step-by-step → Warnings → Checklist
- Career/Framework posts: Scenario → Motivation → Core framework → Breakdown → Role-specific guidance
- Personal Reflection posts: Milestone → Themed sections → Growth narrative → Forward-looking perspective

**Key Patterns:**
- Opens with direct problem statement or personal context
- Uses conversational transitions: "So here's what I think about...", "Here's the thing:", "Let's look at..."
- Acknowledges limitations: "This isn't perfect, but...", "I don't want this to be condemning..."
- Provides actionable elements (don't just leave readers with theory)
- Addresses multiple perspectives when relevant
